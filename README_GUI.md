# DocPortal Desktop 🚀

## 快速开始

### 方式一：命令行启动
```bash
python gui.py
```

### 方式二：从命令行打开指定文件夹
```bash
python gui.py "C:\你的文件夹路径"
```

### 方式三：CLI 门户模式
```bash
python generate.py "C:\你的文件夹" --portal -o ./portal_output
```

---

## 功能特性

### 📂 文件夹选择
- **点击浏览**：点击蓝色区域或「浏览文件夹」按钮
- **拖拽支持**：如果安装了 `tkinterdnd2`，支持直接拖入文件夹
- **快捷键**：`Ctrl + O` 打开文件夹

### 📋 文件列表
- 自动扫描文件夹中所有文件
- 显示文件名、大小、是否支持解析
- 支持与不支持的文档一目了然

### ⚙️ 输出设置
| 设置项 | 说明 |
|--------|------|
| **输出模式** | 选择「单文件 HTML」（传统模式）或「知识门户」（门户模式） |
| **输出路径** | 传统模式：HTML 文件路径；门户模式：输出目录路径 |
| **HTML 文件名** | 传统模式下自定义输出文件名 |
| **最大字符数** | 传统模式：控制输出总字符数（默认 50000），留空=不限 |
| **每页最大字符数** | 门户模式下每个文档页面的字符数上限（默认 8000） |
| **包含不支持文件** | 勾选后在首页中标记不支持的文档 |

### 🚀 一键生成
- 点击「一键生成 HTML」开始解析
- 进度条实时显示
- 完成后弹出详细报告
- 可选择打开文件所在文件夹

### ⌨️ 快捷键
| 快捷键 | 功能 |
|--------|------|
| `Ctrl + O` | 打开文件夹 |
| `Ctrl + G` | 生成 HTML |
| `Ctrl + S` | 选择输出路径 |
| `Esc` | 退出程序 |

## 🏛️ 知识门户模式

DocPortal GUI 支持两种输出模式：

### 传统模式（单文件 HTML）
生成单个大 HTML 文件，适合直接喂给 LLM（Claude / ChatGPT 等）。

**输出设置建议**：
- 输出路径：指向一个 `.html` 文件（如 `output.html`）
- 最大字符数：根据 LLM 上下文窗口设置（如 50000）

### 🆕 知识门户模式
生成可搜索的分页知识门户，推荐在 Edge Copilot 中使用。

**输出设置建议**：
- 输出模式：选择「知识门户」
- 输出路径：指向一个目录（如 `./portal_output`）
- 每页最大字符数：默认 8000，确保 Copilot 能完整读取每页

**门户模式特点**：
- 📑 每个文档生成独立 HTML 页面
- 🔍 首页带搜索框，可实时过滤文档
- 🏷️ 自动提取关键词标签，形成关键词云（含中英文停用词过滤）
- 📇 文档卡片展示：文件名、大小、预览摘要、标签、文件时间
- 🌙 暗黑模式：自动跟随系统颜色主题
- 🖨️ 打印友好：页面内置打印样式
- 🔗 面包屑导航：文档页顶部显示「首页 > 文档名」
- 📋 一键复制全文：每个文档页均有复制按钮
- 🕐 元数据展示：显示文件修改时间、创建时间

## 💡 使用技巧

### 在 Edge Copilot 中阅读门户知识库
1. 使用门户模式生成知识门户
2. 双击 `index.html` 在 Edge 浏览器中打开
3. 搜索关键词找到目标文档
4. 点击文档标题进入详情页
5. 按 `Ctrl+Shift+.` 唤醒 Edge Copilot
6. Copilot 会自动读取当前页面内容，可直接提问

### 生成后的操作
| 操作 | 说明 |
|------|------|
| **查看结果** | 点击「打开文件所在位置」浏览输出文件 |
| **直接阅读** | 双击 HTML 文件在浏览器打开 |
| **喂给 AI** | 将 HTML 文件内容粘贴或上传给 Claude / ChatGPT |

## 输出 HTML 特点

### 传统模式输出
- ✅ 文件头部汇总（文件名、字符数、导出时间）
- ✅ 每个文件独立 `<article>` 区块
- ✅ 文件大小和内容字符数标注
- ✅ 不支持的文档自动标记 `⏭️`
- ✅ 现代简洁 UI 样式，浏览器直接打开即可阅读
- ✅ 超出字符限制自动截断并提示

### 门户模式输出结构
```
output_dir/
├── index.html          ← 首页入口（搜索框 + 关键词云 + 文档卡片列表）
└── docs/
    ├── 技术文档_需求说明书_pdf.html   ← 每个文档独立页面
    ├── 报告_2024年度总结_docx.html
    └── ...
```

## 📦 支持的格式

| 格式 | 说明 |
|------|------|
| TXT / MD / HTML / JSON / XML / CSV / YAML / TOML / INI / LOG / CFG | 文本文件自动识别，支持 UTF-8/GBK/Latin-1 编码 |
| PDF | 基于 pdfminer.six |
| DOCX (Word) | 基于 python-docx（不支持旧版 .doc） |
| PPTX (PowerPoint) | 基于 python-pptx（不支持旧版 .ppt） |
| XLSX (Excel) | 基于 openpyxl（不支持旧版 .xls） |
| 其他（.exe, .zip, .png 等） | 自动静默跳过，不报错 |

> ⚠️ **旧版 Office 格式（.doc / .ppt / .xls）**：建议用 Office 或 WPS 另存为新版格式后再使用。

## 系统要求

- **操作系统**：Windows / macOS / Linux
- **Python**：3.8 及以上版本
- **依赖**：首次运行自动安装（`requirements.txt`）

## 技术栈

- **UI 框架**：tkinter + ttk（Python 标准库）
- **文档解析**：pdfminer.six / python-docx / python-pptx / openpyxl
- **门户生成**：纯 Python + 原生 HTML/CSS/JS，无外部依赖
